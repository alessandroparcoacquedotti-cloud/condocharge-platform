# Screenshots

This folder tracks the required product screenshots for repository presentation, release packaging, and demo support.

## Required Screenshots

- Login
- Resident station status
- Push notifications
- Telegram
- Admin dashboard
- Health dashboard
- Billing
- Users/RFID

## Capture Guidelines

- Use demo data only
- Exclude secrets, tokens, private database paths, and real personal data
- Keep browser size consistent across captures
- Prefer one clean hero screenshot per feature
- Use Italian labels where they are user-facing in the product
