# Demo Walkthrough

This walkthrough is the recommended sequence for a condominium administrator demo.

The flow is intentionally operational: it starts from the admin side, shows how charging gets attributed and billed, and ends in the resident portal.

## 1. Admin Login

- Sign in as the condominium administrator
- Explain that Condo Charge is scoped per condominium and role
- Highlight that admins see resident, billing, and reconciliation workflows that residents do not

## 2. Dashboard

- Open the admin dashboard
- Show the overall building charging summary
- Point out station visibility, recent usage, and operational status

## 3. Create Resident

- Navigate to the resident-management area
- Create a new resident profile for a realistic unit, such as `Lucia Ferraro` in `C-03`
- Explain that resident onboarding is part of the operational workflow, not a separate back-office system

## 4. Assign RFID

- Open RFID assignment for the resident list
- Show how an RFID card is mapped to the correct resident
- Emphasize that session attribution depends on this mapping

## 5. Import Charging Sessions

- Show imported charging sessions from the station integration layer
- Explain that charger exports or synced sessions become billable usage only after attribution
- Use one or two realistic sessions to show timing, energy consumed, and linked RFID user

## 6. Generate Billing Period

- Open billing periods
- Create or review the monthly period, for example `June 2026`
- Explain that the period snapshots the condominium tariff and groups eligible sessions

## 7. Produce Statements

- Generate statements for the selected billing period
- Open a resident statement detail view
- Show:
  - statement number
  - payment reference
  - energy total
  - amount due
  - PDF export

## 8. Resident Login

- Sign out from the admin account
- Log in as a resident such as `Giulia Conti`
- Explain that the resident view is intentionally narrower and focused on transparency rather than administration

## 9. Resident Consumption View

- Open the resident dashboard or sessions page
- Show the resident’s own charging history only
- Connect the visible sessions to the billing statement shown earlier
- If available in the demo flow, open the resident billing page to show statement visibility and payment status

## Suggested Narration

- Shared chargers create a billing and attribution problem for condominiums
- Condo Charge links usage to residents through RFID mapping
- Admins can close a monthly billing cycle without manual spreadsheets
- Reconciliation and payment tracking turn charger usage into an auditable process
- Residents get visibility into their own consumption and charges without seeing anyone else’s data
