## Recommended README Image Placements

When screenshots are ready, prefer a small number of strong images over a large gallery.

1. Add one hero screenshot directly under the opening project summary.
2. Add one admin workflow screenshot near the `Product Summary` or `Architecture` section.
3. Add one resident-facing screenshot near the `Screenshots` section.
4. Keep detailed screenshot guidance in `docs/SCREENSHOTS_CHECKLIST.md`.

## Image Placement Notes

- Store public screenshots under `docs/images/`
- Use filenames like `hero-dashboard.png`, `admin-billing.png`, and `resident-dashboard.png`
- Avoid real names, hostnames, email addresses, payment references, or pilot infrastructure details
- Keep browser size and zoom consistent across all images
